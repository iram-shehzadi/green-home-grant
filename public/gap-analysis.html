4. Risks, blockers & accessibility concerns

🟡 Duplicate / competing CSS (maintenance risk)

main.jsx imports the full govuk-frontend.min.css and App.css hand-rolls its own .govuk-* rules. They overlap (summary-list, radios, buttons). govuk-frontend loads second so it usually wins, but the two can fight. Decide on one source of truth — ideally lean on govuk-frontend and trim App.css to genuine extras.

🟡 npm install required after pulling

The govuk-frontend dependency was added recently — pulling without npm install throws "Failed to resolve import". Team rule: pull → npm install → npm run dev.

🟡 Accessibility claims need evidence

Criteria 10/11/12 are ✅ on code inspection, but a WCAG 2.2 AA submission should demonstrate it: a keyboard-only walkthrough, a 320px screenshot, and a contrast-checker result. Using GOV.UK Frontend components correctly is your strongest defence.

🟢 No functional blockers

The full journey (start → 5 questions → check answers → result) works, validation fires, and the eligibility logic is tested. Nothing is broken.
